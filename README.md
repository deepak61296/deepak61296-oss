# Deepak Popli — Open Source Contributions

Hi, I’m Deepak. I enjoy building useful things on my laptop, often late at night. One of those things is contributing to the open-source tools I use myself.

While using tools such as SITL, autotest, and MAVProxy in my day-to-day work, I sometimes find open issues or bugs that I can reproduce and fix. This repository is a small record of my contributions to the ArduPilot ecosystem.


## What I have learned so far

- Understand and reproduce the bug before trying to fix it.
- Keep changes and commit messages small, clear, and easy to review.
- Test behavior changes in SITL or with autotests whenever possible.
- Never push code I do not understand or have not verified line by line.
- Explain the problem, the fix, and the testing clearly in every pull request.
- Respect project guidelines and the reviewers’ time.
